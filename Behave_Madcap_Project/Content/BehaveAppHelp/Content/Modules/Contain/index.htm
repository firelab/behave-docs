Contain

behaveplus:contain:help

This module adds the containment model to the Surface module. The containment model allows users to game out the resource complement needed to contain a spot fire. The spot fire can be a spot fire adjacent to a prescribed fire unit or an initial attack fire.

Assumptions of the Containment Model

  1. Fire is elliptically shaped.

  2. The fuel types do not change along the fire perimeter.

  3. Each resource constructs line equally along both flanks of the fire.

  4. Each resource will have the same production rate for the entire duration of the shift.

Most users will benefit by first calculating the total line production rate needed to contain the fire. While a future version of Behave will hopefully allow this approach, users can currently calculate this by selecting a broad range of inputs for the resource line production rate. In the following example, a resource line production rate of 10-100 ch/h with steps of 10 ch/h was used.

There are two outputs to focus on.

  • First, the predicted flame length is less than 2 ft so any resource can be used, and a line construction offset is not needed.

  • Second, a total line production rate between 30-40 ch/h is needed to contain the fire. You're looking for that breakpoint between "Withdrawn" and "Contained". By returning to the Suppression tab in the Behave inputs, a range from 30-40 ch/h with a step of 1 can be used as the input for resource line production rate to determine the total line production rate for these conditions.
